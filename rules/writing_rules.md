# 通常投稿 執筆ルール

インスタグラムアカウント：https://www.instagram.com/hana.moja_/
店のコンセプト：「花とグリーン、どちらも主役級。特別なかけあわせ。」

---

## 1. 全体構成（3〜4段落 + フッター + ハッシュタグ）

投稿本文は以下の順序で構成する。

### 第1段落：キャッチコピー（1行）

- **体言止め**で、印象的かつ詩的な一文にする。
- 句点「。」で終える。
- テーマの花・グリーンの個性と、投稿全体のトーンを凝縮する。
- 例：「手のひらサイズのちょこんと早春。球根と苔の芽吹きアレンジメント。」
- **キャッチコピーは「予告編」として機能させる**：花材名を具体列挙すると第2段落の花材描写と内容が被って冗長になる。具体列挙する場合は第2段落を全体感から入る書き出しに切り替えるか、キャッチを抽象寄り（色のトーン・テーマ）にして役割分担を明確にする。
- **3要素を並列に並べる場合は粒度を揃える**：×「赤バラ、白ダリア、ピンクの彩り」（具体物と抽象が混在）／○「赤バラ、白ダリア、ピンクのカーネーション」または「赤と白とピンク」。
- **対比型キャッチでは対比軸に「異なる動詞」を使う**：「贈る花、寄り添う花。」のように動詞を変えると対比が立つ。×「母へ手渡す花、自分へも手渡す花。」のように同じ動詞を繰り返すと対比が弱まり、ただの言い換えに見える。対比の対象（贈る相手 / 自分など）と、それぞれに対応する動作のニュアンスを別の動詞で表現する。
- **キャッチコピーは複数案を比較検討してから選ぶ**：1案で確定させず、5〜6案を出して短さ・対比軸の有無・テーマ宣言の強さ・呼びかけ調 vs 体言止め純度などのバリエーションで比較する。複数案を並べて比較することで、ただの第一案では気づけない磨き込みポイント（動詞の重複、語尾の冗長さ、対比の弱さ）が見える。

### 第2段落：主役の花・グリーンの魅力

- トップ写真に写っている主役の花やグリーンを具体的に紹介する。
- **品種名**を正式に記載する（例: スイートピー、チューリップ"ホワイトフラッグ"）。
- **色・質感・形・ライン**など、五感に訴える描写を入れる。
- 花言葉やその花の特性など、知識的な要素を入れてよい。
- 1枚目の写真の内容と紐づけて描写する（「写真1枚目の…」「1枚目の花束には…」）。
- **キャッチからの接続を硬くしない**：キャッチを体言止めで締めた直後に「〜です。」と説明文を続けるとリズムが急にカクッと硬くなる。本文1行目を体言止め（例：「母の日が近づいたMojaの店頭。」）にして、キャッチの余韻を引き継ぐ選択肢を持つ。

### 第3段落：かけあわせの魅力 / テーマの深掘り

- 複数の花とグリーンの**組み合わせ（かけあわせ）**の妙を伝える。
- コンセプトである「花とグリーン、どちらも主役級。特別なかけあわせ。」を体現するパート。
- 以下のいずれかの切り口で書く：
  - 組み合わせによって生まれる**コントラスト・調和・奥行き**
  - 技術的なこだわりや制作意図
  - テーマに関する深い知識やストーリー
  - 花の品種・咲き方のバリエーション紹介（知識共有型）
- テーマによっては、第2段落でアレンジメント全体の描写をまとめ、第3段落で花の知識共有（品種の多彩さ、開花による変化など）に充てる構成も有効。
- **段落間のつながり**、および**段落内で話題が二つ並ぶ場合のつながり**を意識する。視点を自然にスライドさせる（例：品種間の多様さ → 一輪の中での開花変化、一輪の表情の幅 → 束ねたときの色合わせの豊かさ）。話題が転換するときは橋渡しの一文を必ず挟む。
- **同じ構造の列挙を連続させない**：たとえば2行目で「A、B、C」と3要素を列挙したら、3行目は要約・拡張の一文で受ける。列挙を続けるとリズムが単調になり、読み手が疲れる。
- **テーマの花を限定しすぎない**：「赤バラ」と限定するより「バラ」全般に視点を広げた方が、店の品揃え訴求になり知識共有の幅も語りやすい（写真に写る具体は第2段落で描写済みなので、第3段落では抽象に広げてもよい）。
- **キャッチコピーの語句を本文で呼応させる**：キャッチの主要語（特に対比動詞・キーワード）を、第3段落の中盤と第4段落の来店誘導文に1〜2回再登場させると、投稿全体の一貫性と「予告編→本編→締め」の流れが整う。例：キャッチ「贈る花、寄り添う花。」→第3段落「誰かに贈る花も、自分の暮らしに寄り添う花も〜」→第4段落「贈る花と寄り添う花を一緒に選びましょう」。
- 🌱 のような**絵文字は段落末尾に1つだけ**使ってよい。

### 第4段落：利用シーン提案 + 来店誘導

- 具体的な利用シーン（ギフト、イベント、ご自宅用など）を提案する。
- **季節行事**（成人の日、卒業・入学、母の日など）があれば自然に関連づける。
- 「ぜひMojaで〜」「ぜひ実物で〜」のように来店を促す。
- **括弧書きの補足**で店主の人柄や小ネタを添えてよい（例：「（ちなみに〜ですが、良質な○○も入りました）」）。
- **本文の主軸メッセージとは別の話題（別商品の入荷告知など）の括弧書きは、本文末尾の `(*^^*)` で締めた後ろに、独立した段落として分離する。** 本文の流れと余韻を断ち切らないため。
- **別商品の括弧書き告知は、参照と在庫幅を明示する**：（1）写真がある場合は「写真3枚目、〜」と参照を明示し読者がすぐ確認できるようにする、（2）複数品種があるなら「**各種**〜」と入れて在庫の幅を示唆、（3）特徴は「と」（限定）より「や」（など）で開く（×「くすみピンクと整ったロゼット咲き」／○「くすみピンクや整ったロゼット咲き」）。
- 特定の花材への**具体的な五感の体験提案**（「触れてみてほしい」「香りを確かめてほしい」等）も効果的。
- **対比ペアの語彙は段落間で揃える**：第3段落で「アレンジメント／一輪」と書いたら、第4段落でも「アレンジメント／一輪」と統一する。×第3段落「お母さんへ大きめのアレンジメントを、自分用には小さな一輪を」／第4段落「お母さんへの一束も、自分のための一輪も」のように贈り物の単位（アレンジメント／ブーケ／花束／一束 等）を変えると、読者が「同じものか別のものか」と迷う。対比の片側だけ語彙を変えるのは特に避ける。
- **`(*^^*)`** で文末を締める（毎回必須）。

### ハッシュタグ

- フッターの後に1行で記載する。
- **5個**とする。
- 必ず含めるもの：`#藤沢花屋` `#湘南花屋`
- テーマに応じて追加：花の名前、シーン（`#卒業祝い`）、特徴（`#花のある暮らし` `#フラワーギフト`）。
- **ニッチ・マニア層に刺さるタグを優先する。** 汎用的すぎるタグ（`#スワッグ` `#花束` `#アレンジメント`）は埋もれやすいため避け、花材名（`#バンクシア` `#プロテア`）やジャンル×形態の掛け合わせ（`#ネイティブフラワースワッグ`）で絞る。

## 2. 文体・トーンのルール

- **です・ます調**を基本とする。
- **体言止め**をキャッチコピーだけでなく、段落内でも効果的に使い、テンポを出す。
- **比喩・感覚的表現**を積極的に使う（「まるでドレスのような華やかさ」「ふわっと空間を作る」など）。
- **五感に訴える描写**を重視する：色あい、質感（ふわふわ、しっとり、瑞々しい）、香り、形・ラインなど。
- 堅すぎず、柔らかく知的な語り口。**店主が語りかけるような親しみやすさ**を保つ。
- 過度な装飾語や感嘆を避け、**品のある抑制の効いたトーン**にする。
- **修飾語は最小限に**：一つの名詞句に形容語を重ねない（×「深い赤の整った花姿のバラ」 / ○「整った花姿の赤バラ」）。比喩・装飾語は効果的なポイントに絞り、全体に散らさない。
- **語り手の主観介入を避ける**：「私たちは思います」「〜と思います」のような介入表現は冗長。直接断定する（×「決まった色や形はないと、私たちは思います」 / ○「決まった色や形はありません」）。
- **来店誘導は共同行為のトーンを優先**：「ぜひMojaで〜してください」のような改まった敬語より、「Mojaで、〜を一緒につくりましょう」のように店主と顧客が並走する親しみあるトーンを選ぶ。
- **句点ごとに改行する**：インスタの可読性を優先し、長い文を改行なしで連続させない。一文ごとに改行する。
- **「ですます」を3文以上連続させない**：段落内で「〜です」「〜ます」「〜あります」のような述語が3回以上続くとリズムが平板になる。**体言止めを織り混ぜて緩急をつける**（例：「〜の店頭。／〜が並びます。／〜の大輪。／〜ではありません。／〜広がっています🌱」のように体言止めとです調を交互に配置）。前段の末尾と次段の冒頭が「〜ます。」「〜ます。」と重なる箇所も同様にリズム平板化のリスクがあるため、どちらかを体言止めに切り替える。
- **動詞の擬人化・装飾化を避ける**：「ふっくら重なる」「抱え咲く」のような擬人的・装飾的動詞より、「ふっくらした」「丸い」のような口語的な形容の方が読み流しやすい。装飾は名詞句に絞り、動詞はシンプルに。
- **漢語より和語を優先する**：堅い漢語は親しみある和語に置き換える（×「想いを託す花」／○「想いを乗せた花」）。Mojaの「店主が語りかけるトーン」に整合する。
- **主語の明示と助詞による連結明確化**：倒置・体言止めばかりでなく、「は」で主語を明示すると読みやすさが上がる（×「実はこちらの花束、〜」／○「実はこちらの花束は、〜」）。複合語感を避けるため助詞「の」で連結する（×「ゴールデンウィーク帰省」／○「ゴールデンウィークの帰省」）。
- 絵文字は🌱など自然系のものを**1投稿に1〜2個**に抑える。

## 3. コンテンツの注意事項

- **過去に使った文章をそのまま引用・再利用することは禁止。**文体や構成のみを参考にする。
- **直近1ヶ月の投稿とメッセージ・切り口を重複させない。** 上記「過去引用禁止」は文章レベルの規約。その上位概念として、過去投稿が既に伝えた**主張・切り口・提案ポイント**（例：「お母さんの雰囲気を思い浮かべて色を選ぶ」「定番ピンク以外もある」「花に正解はない」など）の重複も避ける。直近1ヶ月（特に同テーマ＝母の日・成人の日等が連続する場合）の投稿のメッセージ要点を初稿前に内部で整理し、既出主張は別の切り口（知識共有／ライフスタイル提案／技術的こだわり等）に置き換えてから書き始める。文章レベルの引用チェックでは検出できない**メッセージレベルの重複**に注意する。
- 写真に含める要素として、トップ写真において特徴的な要素を「植物の質感」「色あい」「形やライン」「季節性(現在日時を参照)」「組み合わせ」の中からピックアップする。
- **季節性**は現在日時を参照し、旬の花・季節行事と自然に結びつける。
- 品種名が不明な場合はプレースホルダー `{{品種名}}` を置き、確認を求める。
- **投稿文面のファイル形式は `.txt`（プレーンテキスト）とする。** `.md`（マークダウン）では `(*^^*)` などの記号がマークダウン記法として解釈され、意図しない表示になるため。
- **写真の構成を正確に把握すること。** 器が複数ある場合はそれぞれの中身を個別に確認し、花材の組み合わせを思い込みで決めつけない。
- **AIが写真から推測した花材の質感・形状は断定しない。** ステップ1の確認が取れるまでは描写を確定しないこと（例：「網目模様の花序」と書いたが実際は「ふわふわベージュの質感」だった）。
- **テーマのキーワードや表現は事前に検討すること。** 新しい概念や造語を使う場合は、一般的な理解のされ方・誤解の可能性・代替表現を検討し、ユーザーに確認を取ってから執筆に入る。
- **花材の入荷・在庫状況を正確に表現すること。** 投稿日時点で店頭にない花材を「揃っています」等の現在形で書かない。入荷時期が先の場合は「○○ごろから入荷予定」のように時期を明示する。不明な場合は確認を求める。
- **2パターン提案時は書き出しのアプローチを変えること。** 同じ花材名・同じ切り口で始めず、片方は季節感から入る・片方は花材から入る等、第1文の視点を意識的に変える。
- **業界用語・造語の難解さをチェックする。** 「抱え咲く」「咲き重なる」のような花業界用語や造語は、一般読者に伝わるか必ず検討する。難解なら平易な比喩（「カップを描くように咲く」「ふっくら重なる」など）に置き換える。
- **格助詞の正確性に注意する。** 動詞の取る格助詞を確認する（×「花たちで手向ける」／○「花束を手向ける」── 「手向ける」は対象を「を」で受ける）。
- **お供え・弔事に関わる文面では、店主が故人やお客様の関係を推測する言い回しを避ける。** 推測（「よほど〜だったのだろう」）や過去形（「だった」）は、お客様の感情を上書きしたり、喪失の生々しさを刺激する可能性がある。店主は「受け取る側」「お客様のお気持ちに寄り添う側」に立場を引き、お客様の選択（手向けたい／送りたい）そのものへの敬意で締める（×「鮮やかな花束を手向けてもらえるなんて、よほどあたたかい関係だったのだろう」／○「鮮やかな花束を手向けたいというお気持ちに、胸が温まる仕事でした」）。
- **写真は全枚を本文で描写する必要はない。** 複数枚ある場合、トップ写真とテーマに直接関係する写真を優先描写し、参考写真（補足情報用）は本文非言及で構わない。詳細描写は品種誤認・長文化のリスクと引き換えになるため、ユーザーに「この写真は本文で言及するか」を確認する。トップ写真は必須描写。
- **初稿の完成度を引き上げ、TODO往復サイクルを最小化する。** AI初稿 → ユーザーが文章中にTODOタグを埋めて返却 → AI流し込み、という差し戻しは往復コストが高い（数ターン消費する／意図のずれが残る）。これを避けるため、（1）写真確認・テーマ表現・切り口判断・対象読者の幅・告知配置などの**判断系は事前ステップの[Question]/[Answer]で完結**させる、（2）**過去投稿との被りチェック（メッセージレベル）を初稿提案前に自己実施**する、（3）初稿は「ユーザーが微調整指示するだけで完成形に近い」レベルを目標とする。文章本体の構造変更は、ユーザーから具体的修正指示があってから差し替える。
