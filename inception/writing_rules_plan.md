# 執筆ルール抽出 & リクエスト文面ブラッシュアップ 計画

## 概要

過去のインスタグラム投稿（normal_post 4件、calendar_post 1件）を分析し、今後の投稿に反映すべきルールを抽出する。
抽出したルールで `rules/writing_rules.md` を更新し、`rules/request.md` のリクエスト文面もブラッシュアップする。

---

## ステップ

### Phase 1: 分析

- [x] **Step 1: 投稿の構造パターンを整理する**
  - 全5件の投稿を横断的に比較し、共通する構成要素（段落構成・セクション区切り・フッター）を整理する。
  - 「通常投稿(normal_post)」と「カレンダー投稿(calendar_post)」の構成上の違いを明確にする。

- [x] **Step 2: 文体・トーン・表現技法のルールを抽出する**
  - 一貫して使われている文体（です・ます調、体言止め、絵文字使用など）を抽出する。
  - 比喩・五感表現・専門用語のバランスなど、表現技法の傾向を整理する。

- [x] **Step 3: コンテンツ要素のルールを抽出する**
  - 必ず含まれる情報要素（品種名、色・質感の描写、利用シーン提案、来店誘導など）を洗い出す。
  - ハッシュタグの構成ルール（数・カテゴリ）を整理する。

### Phase 2: ルール策定

- [x] **Step 4: 分析結果をルール一覧にまとめる**
  - Phase 1 の分析結果を、カテゴリごとに整理した「執筆ルール一覧」としてドラフトする。
  - 「通常投稿」用ルールと「カレンダー投稿」用ルールを必要に応じて分ける。

  [Question] 既存の `writing_rules.md` にある参考例文（秋のアレンジメントの投稿例）は、更新後も残しますか？それとも、実際に投稿された文例（`/posted` 内のもの）に差し替えますか？
  [Answer] postedに差し替える

  [Question] カレンダー投稿（営業日案内の投稿）用のルールは `writing_rules.md` 内に通常投稿ルールと一緒にまとめますか？それとも別ファイルにしますか？
  [Answer] 別ファイルにする

- [x] **Step 5: `rules/writing_rules.md` を更新する**
  - Step 4 で承認されたルール一覧の内容で `writing_rules.md` を書き換える。

- [x] **Step 5b: `rules/calendar_writing_rules.md` を新規作成する**
  - カレンダー投稿用の執筆ルールを別ファイルとして作成する。

### Phase 3: リクエスト文面ブラッシュアップ

- [x] **Step 6a: `rules/request.md`（通常投稿用）を改善する**
  - 新ルールを反映した、都度のリクエスト時に使うテンプレート文面を作成する。
  - 毎回変わる可変部分（テーマ、写真説明など）を明確にプレースホルダー化する。
  - 参照すべきルールファイルのパスを明記する。

  [Question] 現在の `request.md` は「通常投稿」のリクエストのみですが、「カレンダー投稿」用のリクエストテンプレートも別途作成しますか？
  [Answer] 別途作成する

  [Question] リクエスト文面に「参照すべきルールファイルのパス」を明記する形にしますか？（例: 「以下のルールに従ってください → rules/writing_rules.md」のような指示を含める）
  [Answer] 明記する

- [x] **Step 6b: `rules/calendar_request.md`（カレンダー投稿用）を新規作成する**
  - カレンダー投稿用のリクエストテンプレートを作成する。

### Phase 4: レビュー

- [x] **Step 7: 最終レビュー**
  - 更新した `writing_rules.md`、`calendar_writing_rules.md`、`request.md`、`calendar_request.md` の内容を最終確認し、レビューを依頼する。

---

## 補足

- 各ステップ完了時にこのファイルのチェックボックスを `[x]` に更新します。
- 重要な判断が必要な箇所には `[Question]` / `[Answer]` タグを付けています。回答をいただいてから次のステップに進みます。
